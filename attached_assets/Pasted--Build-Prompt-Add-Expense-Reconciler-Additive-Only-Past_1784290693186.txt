## Product: Expense Reconciler (new, separate feature)

User uploads a receipt photo AND a voice memo explaining the expense. The product reconciles both into a structured, editable, exportable expense entry, flagging any mismatch between what was claimed and what the receipt shows.

## Pipeline — same 3-step structure, build fresh, don't touch the podcast pipeline

**Step 1 — Audio Understanding**
Input: `{ audio_file_url }`
Groq Whisper transcription → short LLM call extracting structured intent.
Output:
```ts
type AudioResult = {
  transcript: string;
  claimed_purpose: string | null;
  claimed_amount: number | null;
  claimed_counterparty: string | null;
  claimed_date: string | null;
  confidence: number;
}
```

**Step 2 — Receipt Understanding**
Input: `{ image_base64 }`
Single Anthropic vision call (direct multimodal call, not an OCR service) extracting structured fields with per-field confidence.
Output:
```ts
type ReceiptResult = {
  vendor: string;
  line_items: { description: string; amount: number }[];
  total: number;
  date: string | null;
  category_guess: string;
  field_confidence: Record<string, number>;
}
```

**Step 3 — Reconciliation** (the real cross-modal reasoning step — both Step 1 and Step 2 outputs go into one Anthropic call together, model explicitly reasons about agreement/disagreement before producing JSON, not a merge)
Input: `{ audio_result: AudioResult, receipt_result: ReceiptResult }`
Output:
```ts
type ReconciledExpense = {
  category: string;
  amount: number;
  vendor: string;
  date: string;
  business_justification: string;
  flags: { field: string; reason: string; severity: "low" | "medium" | "high" }[];
  overall_confidence: number;
}
```

Flags must reflect genuine detected mismatches (amount vs claimed_amount, vendor vs claimed_counterparty, category vs line items) — instruct the model to reason step-by-step before emitting flags, not output a cosmetic confidence number.
